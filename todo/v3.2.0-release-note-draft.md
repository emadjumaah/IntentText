# v3.2.0 Release Note Draft (Pure Rust Core)

Date: `10/03/2026`
Status: `draft`

## Headline

IntentText `v3.2.0` ships a pure-Rust core path for parser and renderer runtime execution.

## What Changes

- Runtime parser/renderer execution is Rust/WASM-first for core flows.
- Theme parity contract is implemented for Rust render path.
- Runtime fallback telemetry is exposed and enforced in CI for theme fallback (`renderer_theme_fallback_to_ts` must remain zero).
- CLI telemetry command is available for migration visibility:
  - `node cli.js telemetry`
  - `node cli.js telemetry --json`
  - `node cli.js telemetry --reset`

## Compatibility Notes

- JavaScript compatibility wrappers remain during cutover where required.
- Existing APIs remain available while TS runtime removal work proceeds through Step 4.

## Migration Notes

- Consumers should treat Rust/WASM output as source-of-truth for parser/renderer behavior.
- Any non-zero fallback telemetry should be treated as a parity blocker.
