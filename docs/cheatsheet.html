IntentText (.it) Cheatsheet

One-page reference for the most common IntentText syntax. Use Download PDF to print/save as PDF.

Playground Spec

Block pattern

[Keyword]: [Content] | [Property]: [Value] | [Property]: [Value]

Keywords are case-insensitive. Properties are optional and open-ended.

File basics

Rule Value
Extension .it
Encoding UTF-8
Line endings LF or CRLF

Core keywords

Keyword Meaning Example
title: Document title / identity title: *Project Dalil* Launch Plan
summary: Short document description summary: Finalizing deployment in _Doha_.
section: Major heading / context section: Team Tasks
sub: Sub-section sub: Technical Details
note: Standalone fact / paragraph note: Witness is 40 yrs old
divider: Visual separator (optional label) divider: End of Section
task: Actionable to-do task: Database migration | owner: Ahmed | due: Sunday
done: Completed task done: Secure the domain | time: 09:00 AM
question: Open question question: Who has the _Master Key_?
link: External URL link: Docs | to: https://dalil.ai/docs
ref: Cross-document reference ref: Related Doc | to: doc.it#section:Tasks
image: Image reference image: Banner | at: assets/banner.png | caption: Launch artwork
headers: Table headers headers: Item | Location | Status
row: Table row (matches headers order) row: Dell Server | Rack 04 | Delivered

Lists

- Unordered list item
- Another item

1. First ordered step
2. Second ordered step

Unordered: lines starting with - or *. Ordered: digit + ..

Inline formatting (WhatsApp-style)

Syntax Meaning Example
*text* Bold title: *Bold Title*
_text_ Italic note: Work in _Doha_
~text~ Strikethrough note: ~Deprecated~
```text``` Inline code note: Saved at ```/logs/cam1```

v1.0: marks are within one line and non-nesting.

Pipe metadata

task: Database migration | owner: Ahmed | due: 2026-03-01
image: Team Photo | at: assets/photo.png | caption: Team Day
link: Docs | to: https://example.com | title: Documentation

Split on | . Unknown properties should be preserved.

Multi-line code blocks

code:
```
SELECT *
FROM users
WHERE active = true
```
end:

If code: has no inline content, capture lines until an end: line.

Escaping

note: A literal pipe looks like \|
note: A literal backslash looks like \\

Escaping is applied before pipe splitting.

Quick full example

title: *Project Dalil* Launch Plan
summary: Finalizing the deployment for the AI-Agent hub in _Doha_.

section: Logistics & Equipment
headers: Item | Location | Status
row: Dell Server | Rack 04 | Delivered
row: Fiber Cables | Storage | ~Missing~ Ordered

- Set up the environment.

done: Secure the domain name | time: 09:00 AM

divider: End of Technical Sections
